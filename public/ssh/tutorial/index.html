Что такое SSH и зачем он нужен

SSH позволяет подключаться к удалённому серверу и выполнять команды так, будто вы сидите прямо перед ним. Это основной инструмент для:

  • Администрирования серверов
  • Управления VPS и облачными серверами
  • Развёртывания приложений
  • Работы с базами данных
  • Настройки веб-серверов
💡 Важно: SSH-соединение зашифровано, поэтому данные (включая пароли) нельзя перехватить.

Как подключиться к серверу по SSH

Для подключения по SSH вам нужны:

  • IP-адрес сервера — например, 192.168.1.100
  • Порт — по умолчанию 22
  • Логин — обычно root или созданный пользователь
  • Пароль — пароль пользователя
1

Узнайте данные сервера

IP-адрес и порт SSH можно найти:

  • В панели управления хостингом (Timeweb, DigitalOcean, AWS)
  • В письме от хостера при заказе VPS
  • У системного администратора
2

Подключитесь через WebSSH

Самый простой способ — использовать веб-терминал:

  1. Откройте webssh.su
  2. Зарегистрируйтесь или войдите
  3. Нажмите «Новое подключение»
  4. Введите IP, порт, логин и пароль
  5. Нажмите «Подключиться»

SSH на Windows

Способ 1: Windows PowerShell (Windows 10+)

Нажмите Win + X → Windows PowerShell и введите:

ssh username@192.168.1.100

Замените username на ваш логин и IP на адрес сервера.

Способ 2: PuTTY (Windows 7/8)

Скачайте PuTTY с putty.org:

  1. В поле «Host Name» введите IP сервера
  2. Port: 22 (или ваш)
  3. Connection type: SSH
  4. Нажмите «Open»

SSH на macOS и Linux

Откройте Terminal и введите:

ssh username@192.168.1.100

Если SSH на нестандартном порту:

ssh -p 2222 username@192.168.1.100

SSH с телефона

Управлять сервером можно и с телефона:

  • iOS — Termius, Prompt
  • Android — Termius, JuiceSSH
  • Браузер — webssh.su (работает на любом устройстве)
💡 Совет: WebSSH работает прямо в браузере — не нужно устанавливать приложения.

SSH-ключи вместо пароля

SSH-ключи безопаснее пароля и позволяют входить без ввода пароля.

Создание ключей

# На своём компьютере
ssh-keygen -t ed25519

# Скопировать ключ на сервер
ssh-copy-id username@192.168.1.100

Использование ключей в WebSSH

  1. При создании подключения выберите «SSH-ключ»
  2. Вставьте свой закрытый ключ в поле
  3. Подключайтесь без пароля

Решение проблем

Connection refused

SSH-сервис не запущен:

sudo systemctl start ssh
sudo systemctl enable ssh

Permission denied

Неправильный логин или пароль. Проверьте:

  • Логин (обычно root)
  • Регистр букв в пароле
  • IP-адрес сервера

Connection timed out

Сервер недоступен. Проверьте:

  • Правильный ли IP-адрес
  • Открыт ли порт 22 в файрволе
  • Работает ли сервер
⚠️ Файрвол: Иногда порт 22 заблокирован. Проверьте настройки облачного провайдера (AWS, DigitalOcean и т.д.).
